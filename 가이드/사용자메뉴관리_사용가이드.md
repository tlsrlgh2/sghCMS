# 사용자 메뉴관리 사용 가이드

SGH 청년공간 CMS — 사용자(프론트) 페이지 메뉴를 관리자 화면에서 직접 등록·관리하는 기능 안내

---

## 1. 개요

기존에는 사용자 페이지 상단 메뉴(GNB)가 JSP에 하드코딩되어 있어, 메뉴를 바꾸려면 소스 수정·재배포가 필요했습니다.
이제 **관리자 화면 → 사용자 메뉴관리**에서 메뉴를 직접 추가하고, 각 메뉴를 **콘텐츠 페이지** 또는 **게시판**과 연결할 수 있습니다.

- 메뉴 이름, 유형(폴더/콘텐츠/게시판), 노출 여부, 정렬 순서, 계층(상위 메뉴)을 관리
- "메뉴 먼저 생성 → 콘텐츠/게시판은 나중에 연결" 흐름 지원
- 등록된 메뉴는 사용자 페이지 상단 메뉴에 자동 반영(미등록 시 기존 정적 메뉴로 폴백)

---

## 2. 전체 구조

```
[관리자] 사용자 메뉴관리 화면
   │  메뉴 등록/수정/삭제
   ▼
SGH_USER_MENU (전용 테이블)
   ├ MENU_NM    메뉴 이름
   ├ MENU_TYPE  FOLDER | CONTENT | BOARD
   ├ URL_PATH   /user/page/{경로}.do  (의미있는 URL)
   ├ LINKED_KEY 콘텐츠키 또는 게시판ID (나중에 연결 가능)
   ├ UPPER_ID   상위 메뉴(계층)
   ├ SORT_ORDR  정렬
   └ USE_AT     노출 여부(Y/N)
   │
   ▼
[사용자] 상단 메뉴 자동 구성 → /user/page/{경로}.do 접근
   ├ CONTENT → 페이지 콘텐츠 렌더링
   └ BOARD   → 연결된 게시판 목록으로 이동
```

---

## 3. 사전 준비 (최초 1회)

DB에 테이블과 관리자 메뉴를 등록합니다. (운영 DB에 1회 적용)

1. 테이블 생성
   `SGHCMS/script/ddl/mysql/sghcms_user_menu_DDL_mysql.sql`
2. 관리자 메뉴 등록 (콘텐츠/게시판 그룹에 "사용자 메뉴관리" 추가)
   `SGHCMS/script/dml/mysql/sghcms_user_menu_DML_mysql.sql`
3. 애플리케이션 재기동 후, 관리자 좌측 메뉴에 **사용자 메뉴관리**가 보이는지 확인

---

## 4. 관리자 화면 진입

- 관리자 로그인 → 좌측 메뉴 **콘텐츠/게시판 ▸ 사용자 메뉴관리**
- 직접 URL: `/admin/content/userMenu/list.do`
- 화면 구성: 좌측 = 등록된 메뉴 목록, 우측 = 등록/수정 폼

---

## 5. 메뉴 추가하기

우측 "새 메뉴 추가" 폼에서 입력 후 **저장**.

| 항목 | 설명 |
|---|---|
| 메뉴 이름 | 사용자에게 보일 이름 (예: 청년정책) |
| 메뉴 유형 | 폴더 / 콘텐츠 페이지 / 게시판 중 선택 |
| 상위 메뉴 | 최상위 또는 특정 폴더 하위로 배치 |
| URL 경로 | 콘텐츠·게시판 유형에서 필수. 영소문자·숫자·하이픈 (예: policy) |
| 정렬 순서 | 같은 레벨 내 표시 순서 (작을수록 앞) |
| 노출 여부 | 노출/숨김 |

> 유형이 **폴더**이면 URL·연결 입력이 숨겨집니다(하위 메뉴를 묶는 그룹용).

---

## 6. 유형별 연결 방법

### 6-1. 콘텐츠 페이지
- 유형 = **콘텐츠 페이지** 선택 후 저장 → 콘텐츠가 메뉴에 자동 연결됨
- 목록에서 해당 메뉴의 **[콘텐츠 편집]** 버튼 클릭
- **CKEditor**로 본문 작성/수정 후 저장
- 페이지 키는 메뉴에 자동 부여(`menu_{id}`)되어 직접 다룰 필요 없음
- 작성 전에는 사용자 화면에 "준비 중" 표시, 메뉴 삭제 시 콘텐츠도 함께 삭제

### 6-2. 게시판
- 유형 = **게시판** 선택
- "연결 게시판" 드롭다운에서 게시판 선택 (게시판 관리에서 미리 생성)
- 사용자가 메뉴 클릭 시 해당 게시판 목록으로 이동

---

## 7. 사용자 화면 확인

- 등록·노출된 메뉴는 사용자 페이지 상단 메뉴에 자동 표시
- 접근 URL 규칙: `/user/page/{URL경로}.do` (예: `/user/page/policy.do`)
- 폴더 메뉴는 드롭다운 그룹으로, 하위 메뉴가 링크로 표시

---

## 8. 보안 및 주의사항

- **URL 경로**는 영소문자·숫자·하이픈만 허용 (경로 변조·스크립트 삽입 차단)
- **게시판 ID**는 영문·숫자·언더스코어만 허용 후 인코딩하여 이동 (리다이렉트 인젝션 차단)
- **콘텐츠 본문 HTML**은 저장 시 OWASP 새니타이저로 정제되어 저장
- **URL 경로 중복** 불가(유니크), 자기 자신·순환 상위 지정 차단
- **하위 메뉴가 있는 폴더**는 삭제 불가 → 하위 정리 후 삭제
- 모든 DB 접근은 MyBatis 파라미터 바인딩(`#{}`)으로 SQL 인젝션 방지

---

## 9. 자주 묻는 질문

**Q. 메뉴를 만들었는데 사용자 화면에 안 보여요.**
A. 노출 여부가 "노출"인지, 콘텐츠/게시판 연결 또는 URL 경로가 설정됐는지 확인하세요. 폴더는 하위 메뉴가 있어야 의미가 있습니다.

**Q. 기존 하드코딩 메뉴는 어떻게 되나요?**
A. 등록된 사용자 메뉴가 하나도 없으면 기존 정적 메뉴로 폴백됩니다. 메뉴를 등록하기 시작하면 DB 메뉴가 우선합니다. 점진적으로 이전하세요.

**Q. URL을 바꾸면 기존 링크는요?**
A. URL 경로 변경 시 이전 경로로의 접근은 메인으로 리다이렉트됩니다. 외부 공유 링크가 있으면 주의하세요.
